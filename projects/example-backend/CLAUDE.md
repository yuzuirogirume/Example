# 条例: バックエンドプロジェクト (example-backend)

> **継承関係**: この条例はルートの `CLAUDE.md`（憲法）に従います。
> 憲法に違反する条項を定めることはできません。

---

## プロジェクト概要

- **目的**: REST API の提供
- **技術スタック**: TypeScript, Node.js 20, Express
- **担当範囲**: `src/` 配下のサーバーサイドコード全般

---

## 第1条　技術スタックのルール

### 1-1. 使用言語・バージョン

TypeScript 5.x を使用する。Node.js は 20 LTS を使用する。

### 1-2. パッケージ管理

npm を使用する。`package-lock.json` を必ずコミットする。

---

## 第2条　API 設計規約

### 2-1. エンドポイント命名

REST の慣習に従う。リソース名は複数形・スネークケースで定義する。
例: `/api/v1/user_profiles`

### 2-2. レスポンス形式

全エンドポイントのレスポンスは以下の形式に統一する:
```json
{ "data": {}, "error": null }
```

### 2-3. バリデーション

外部入力（リクエストボディ・クエリパラメータ）は
必ず zod でバリデーションする（憲法 第1章 1-1 セキュリティ原則を継承）。

---

## 第3条　データベース規約

### 3-1. ORM

Prisma を使用する。生 SQL は原則禁止。
パフォーマンス上の理由で生 SQL が必要な場合はレビューを必須とする。

### 3-2. マイグレーション

スキーマ変更は必ず `prisma migrate` を使用する。
本番 DB への直接変更は禁止。

---

## 第4条　テスト規約

### 4-1. テストフレームワーク

Jest を使用する。

### 4-2. テスト対象

- 全エンドポイントに統合テストを書く
- ビジネスロジックには単体テストを書く
- DB アクセスを伴うテストはトランザクションロールバックを使用する
