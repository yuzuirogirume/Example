# 条例: フロントエンドプロジェクト (example-frontend)

> **継承関係**: この条例はルートの `CLAUDE.md`（憲法）に従います。
> 憲法に違反する条項を定めることはできません。

---

## プロジェクト概要

- **目的**: ユーザー向け Web UI の提供
- **技術スタック**: TypeScript, React 18, Vite
- **担当範囲**: `src/` 配下の全フロントエンドコード

---

## 第1条　技術スタックのルール

### 1-1. 使用言語・バージョン

TypeScript 5.x を使用する。`any` 型の使用は原則禁止。
やむを得ない場合は `// eslint-disable-next-line` でコメント理由を明記する。

### 1-2. パッケージ管理

npm を使用する。`package-lock.json` を必ずコミットする。

---

## 第2条　コーディング規約

### 2-1. コンポーネント設計

- 1ファイル1コンポーネントを原則とする
- Props の型は必ず明示的に定義する
- コンポーネントは `src/components/` に配置する

### 2-2. 状態管理

グローバル状態は React Context または Zustand を使用する。
Redux は使用しない。

---

## 第3条　テスト規約

### 3-1. テストフレームワーク

Vitest + React Testing Library を使用する。

### 3-2. テスト対象

- ユーザー操作を伴うコンポーネントには必ずテストを書く
- ユーティリティ関数は単体テストを書く

---

## 第4条　スタイリング

### 4-1. CSS

Tailwind CSS を使用する。インラインスタイルは使用しない。

### 4-2. デザイントークン

色・スペーシング・フォントサイズは `tailwind.config.ts` で定義した
デザイントークンを使用する。ハードコードは禁止。
