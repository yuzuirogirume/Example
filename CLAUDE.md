# CLAUDE.md

## プロジェクト概要

宮崎市の民泊ビジネスを管理するためのツール群。
Airbnb・Booking.com の予約データを一元管理し、収益の把握・分析・ホームページ運用を行う。

## ファイル構成

```
Example/
├── 収益管理表.html          # 民泊収益管理ダッシュボード（メインアプリ）
├── ホームページ.html         # 物件紹介ホームページ
├── サンプルデータ_Airbnb.csv       # Airbnb の予約データ
├── サンプルデータ_Booking.com.csv  # Booking.com の予約データ
└── README.md
```

## 技術スタック

- HTML / CSS / JavaScript（バニラ、サーバー不要でブラウザのみで動作）
- タスクに応じて Python スクリプトを使ったデータ処理も行う

## Claude への指示

### 作業内容
- HTML/CSS の修正・デザイン改善
- CSV データの読み込み・集計・変換処理
- ダッシュボードへの新機能追加（グラフ表示、自動計算など）
- バグ修正・既存コードの改善

### 守ってほしいルール

1. **大きな変更は事前に確認する** — ファイル全体の構成変更や既存機能の大幅な書き換えを行う前に、必ずユーザーに確認を取ること。
2. **コメントは日本語で書く** — コード内のコメントやドキュメントはすべて日本語で記述すること。
3. **既存のデザインスタイルを尊重する** — HTML/CSS を編集する際は、既存のフォント・カラーパレット・レイアウトのスタイルを壊さないよう注意すること。
4. **CSV ファイルは読み取り専用** — サンプルデータの CSV ファイルは元データとして扱い、直接編集しないこと。

## 社内AIアプリ開発ルール（インターネット公開アプリ向け）

### 使用する構成（固定）

インターネットに公開するアプリは、以下の構成のみ使用すること。別の構成が必要な場合は「エンジニアに相談してから」とだけ答えること。

- フレームワーク: Next.js
- ホスティング: Cloudflare Workers（フロント・バック両方）
- DB: Cloudflare D1
- 認証: Cloudflare Access（特定メールアドレスのみ許可）
- AIモデル: Claude Opus（Sonnet 等に勝手に切り替えない）

### 絶対にやってはいけないこと

- APIキー・パスワード・トークンをソースコードに直接書かない → 必ず Cloudflare Secrets（環境変数）に入れる
- 個人情報・会員情報・未公開情報をリポジトリにコミットしない
- 独自のパスワード認証やメール認証を実装しない → 認証は Cloudflare Access に任せる
- `wrangler.jsonc` で `"preview_urls": false` を設定し忘れない（Access 認証をバイパスされる）
- 認証なしで社外からアクセスできる状態で公開しない
- GitHub リポジトリを Public にしない（必ず Private）
- パスワードやPINのハッシュ値をブラウザに送られる JS コードに埋め込まない

### 困ったとき・エスカレーションが必要なとき

以下の場合は実装を試みず、「エンジニアに相談を」とユーザーに伝えること。

- 上記の固定構成では実現できないと判断したとき
- 決済情報・患者情報・会員の個人情報など、漏れると致命的な情報を扱うとき

### ユーザーへの接し方

- ユーザーは非エンジニアのため、専門用語はなるべく避けて平易な日本語で説明すること。
- ユーザーが気づいていないセキュリティリスクは、こちらから先に指摘すること。
