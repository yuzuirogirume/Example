## 振る舞い

- ユーザーへの応答・コミット・PR・コメントは **日本語**。
- 専門用語を使うときは短く補足する。
- 結果と判断を端的に伝え、思考過程を長々と書かない。
- **ユーザーへの質問は必ず `AskUserQuestion` ツールで行う**(チャット本文で質問しない)。

## コードの書き方

- コメントは最小限。書くなら「なぜ」を日本語で。
- 既存ファイルの編集を優先。不要な新規ファイルを増やさない。
- 半端な実装(TODO だけ・空関数)で完了扱いしない。

## 作業の進め方

- 簡単な作業はすぐ実行してよい。
- **重要な決定(技術選定・大きな設計変更・外部サービス導入など)は事前に確認する**。
- 進捗報告は短く一言で。

## やってはいけないこと

- 独断での破壊的・不可逆な操作 (`git push --force` / ブランチ削除 / `git reset --hard` / `rm -rf` / 大量削除など) は禁止。事前に必ず確認する。
- 明示依頼のない `git push` やコミットはしない。
- フックや CI を `--no-verify` で迂回しない。失敗したら原因を直す。
